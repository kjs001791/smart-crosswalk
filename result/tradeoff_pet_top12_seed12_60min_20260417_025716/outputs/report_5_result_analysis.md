# 스마트 횡단보도 SUMO 시뮬레이션 결과 분석

## 1. 시뮬레이션 설명

이번 실행은 서울 중구 내 위험도 상위 12개 횡단보도를 대상으로 일반 신호(`baseline`)와 스마트 횡단보도 신호(`smart`)를 비교한 SUMO 기반 미시 교통 시뮬레이션이다. 분석 목적은 스마트 신호가 보행자 안전 지표를 개선하는지, 그리고 그 개선이 차량 지체나 대기행렬 증가라는 교통 비용을 동반하는지 확인하는 것이다.

실제 실행 설정은 `run_metadata.json`과 `report_preset.json` 기준으로 다음과 같다.

| 항목 | 값 |
|---|---:|
| 분석 대상 | 위험도 상위 12개 횡단보도 |
| 시나리오 | baseline, smart |
| seed | 42~53, 총 12개 |
| 분석 시간 | 3600초, 60분 |
| 워밍업 | 300초, 5분 |
| TraCI step length | 0.5초 |
| 수요 프로파일 | `seoul_commute_peak` |
| 네트워크 생성 방식 | 기존 SUMO 네트워크 재사용 |
| 총 SUMO 실행 수 | 288회 |
| 실패 케이스 | 없음 |

참고로 개별 `sumocfg` 파일의 `<step-length>`에는 수요 생성 단계 기본값인 1.0초가 기록되어 있지만, 실제 metric 수집 실행은 `run_metadata.json`의 `traci_step_length=0.5` 값을 TraCI/SUMO 실행 인자로 넘기는 방식으로 수행되었다. 따라서 본 보고서는 실제 실행 인자인 0.5초를 기준으로 해석한다.

총 실행 수는 다음과 같이 계산된다.

```text
12개 횡단보도 x 12개 seed x 2개 시나리오 = 288회 SUMO 실행
```

분석 대상은 사고건수, 노인비율, 차로수, 제한속도 등을 반영한 `risk_score` 상위 12개 지점이다. 상위 후보에는 중림동, 신당동, 회현동, 장충동, 명동 지점이 포함되었다. 위험도 점수 1위는 `23040(중림동, 봉래동2가)`, 2위는 `9494(신당동)`, 3위는 `6122(회현동, 남창동)`이었다.

스마트 신호는 보행 녹색 종료 직전에 횡단보도 위 보행자가 감지되면 보행 시간을 연장하는 방식이다. 본 모델의 주요 신호 설정은 다음과 같다.

| 항목 | 값 |
|---|---:|
| 기본 신호주기 | 120초 |
| 황색 시간 | 4초 |
| 전적색 시간 | 3초 |
| 보행 녹색 산정 | `crossing_length_m / 1.0 + 7초` |
| 횡단보도 길이 산정 | `LANES x 3.5m` |
| 스마트 연장 조건 | 보행 녹색 잔여 10초 이하, 횡단 중 보행자 존재 |
| 연장 시간 | 5초 |
| 최대 연장 | 1회 |
| 센서 false negative | 5% |

사용한 핵심 안전 지표는 두 가지이다. `PET_A_proxy`는 차량이 상충구역 주변 edge를 벗어난 뒤 보행자가 crossing edge에 진입하는 시간차를 이용한 proxy이다. `PET_B_surrogate`는 보행 녹색 종료 시점에 횡단보도 위에 남아 있는 보행자의 잔여 횡단시간과 전적색 시간을 비교한 surrogate 지표이다. 특히 `PET_B_surrogate_severe`는 보행 종료 시점의 잔류 위험을 보는 지표이므로, 현재 스마트 신호 로직의 직접 효과를 가장 잘 반영한다.

중요한 해석상 주의점은 `PET_A_proxy`와 `PET_B_surrogate`가 실제 현장 trajectory pair 기반 PET가 아니라는 점이다. 따라서 본 결과의 PET는 실제 사고위험의 절대값이 아니라 baseline과 smart를 같은 조건에서 비교하기 위한 상대 지표로 해석해야 한다.

실제 실행 과정은 다음 순서로 진행되었다.

1. `T1_accident_crosswalk.csv`와 `T2_crosswalk_features.csv`를 읽고 결측값을 보정했다. `LANES`, `MAX_SPD`, `추정AADT`, `노인비율` 등의 결측은 기본값 또는 중앙값으로 대체되었다.
2. 차로수, 제한속도, 사고건수, 노인비율을 이용해 `risk_score`를 계산했고, 위험도 상위 12개 횡단보도를 후보로 선정했다.
3. 기존에 생성된 SUMO 네트워크를 재사용했다. 이번 실행의 `network_mode`는 `reuse`이고, `skip_networks=true`로 기록되어 있다. 즉 OSM 다운로드와 `netconvert`를 새로 수행하지 않고, 이전 네트워크를 이번 결과 폴더의 `sumo_nets/` 아래에서 재사용했다.
4. 각 횡단보도와 seed별로 차량 수요와 보행자 수요를 생성했다. 차량은 `randomTrips.py` 기반 route를 사용했고, 보행자는 지수분포 도착 간격으로 생성했다.
5. 각 seed마다 동일한 수요 조건으로 baseline과 smart를 모두 실행했다. 따라서 시나리오 차이는 주로 스마트 보행 녹색 연장 로직에서 발생한다.
6. SUMO/TraCI 실행 중 보행자 crossing 진입, 보행 녹색 종료 시점의 잔류 보행자, 차량 누적 대기시간, 접근 차로 정지 차량 수를 수집했다.
7. seed별 결과를 `simulation_results_seed.csv`로 저장하고, 횡단보도별 seed 평균을 `simulation_results.csv`와 `comparison_table.csv`로 집계했다.

실행 결과 파일 기준으로, `simulation_results_seed.csv`에는 288개 seed-시나리오 결과 행이 있고, `simulation_results.csv`에는 12개 지점 x 2개 시나리오의 seed 평균 24개 행이 있다. `failed_cases.csv`는 생성되지 않았으므로, 기록된 실패 케이스는 없다.

수요 조건은 출퇴근 첨두를 가정한 `seoul_commute_peak` 프로파일이다. 이 프로파일은 평균 수요보다 차량과 보행자 수요를 높게 설정한다. 실제 생성된 수요의 평균은 다음과 같다.

## 3. 결과

전체 12개 지점의 seed 평균값을 집계하면, smart 시나리오는 보행 안전 proxy를 개선했고 차량 운영 지표는 거의 변하지 않았다.

| 지표 | baseline | smart | 변화량 | 변화율 |
|---|---:|---:|---:|---:|
| PET_A proxy 고위험 | 1063.33 | 1005.33 | -58.00 | -5.45% |
| PET_B surrogate 고위험 | 907.50 | 666.83 | -240.67 | -26.52% |
| 고령자 미완료 횡단 | 200.25 | 139.92 | -60.33 | -30.13% |
| 차량 평균 지체 합계 | 419.77초 | 419.16초 | -0.61초 | -0.15% |
| 차량 평균 지체 평균 | 34.98초 | 34.93초 | -0.05초 | -0.15% |
| 차량 최대 지체 평균 | 81.58초 | 81.60초 | +0.02초 | +0.03% |
| 평균 대기행렬 | 0.733대 | 0.732대 | -0.000대 | -0.06% |
| 최대 대기행렬 | 3.576대 | 3.576대 | 0.000대 | 0.00% |

`PET_A_proxy`는 5.45% 감소에 그쳤다. 반면 `PET_B_surrogate` 고위험은 26.52%, 고령자 미완료 횡단은 30.13% 감소했다. 이는 스마트 신호가 모든 상충 상황을 균일하게 줄인 것이 아니라, 보행 녹색 종료 시점에 횡단 중인 보행자의 잔류 위험을 줄이는 데 더 직접적으로 작동했음을 의미한다.

지점별로 보면 `PET_B_surrogate` 고위험 이벤트가 발생한 지점은 12개 중 3개뿐이었다. 나머지 9개 지점은 이번 60분 x 12 seed 조건에서 해당 surrogate 이벤트가 0으로 관측되었다. 이는 해당 지점이 안전하다는 뜻이 아니라, 현재 모델과 수요 조건에서 `PET_B_surrogate` 고위험 이벤트가 발생하지 않았다는 뜻이다.

`PET_B_surrogate`가 실제로 발생한 3개 지점의 결과는 다음과 같다.

| 횡단보도ID | 행정동 | baseline PET_B 고위험 | smart PET_B 고위험 | PET_B 감소율 | baseline 고령자 미완료 | smart 고령자 미완료 | 고령자 감소율 | 차량 지체 변화 | 최대 대기행렬 변화 |
|---:|---|---:|---:|---:|---:|---:|---:|---:|---:|
| 9494 | 신당동 | 123.58 | 43.58 | 64.73% | 36.08 | 13.33 | 63.05% | +0.88초 | +0.17대 |
| 8430 | 신당동 | 449.00 | 310.83 | 30.77% | 102.92 | 70.58 | 31.42% | -0.33초 | 0.00대 |
| 71666 | 장충동 | 334.92 | 312.42 | 6.72% | 61.25 | 56.00 | 8.57% | -1.15초 | -0.17대 |

가장 뚜렷한 안전 개선은 `9494(신당동)`에서 나타났다. `PET_B_surrogate` 고위험은 123.58건에서 43.58건으로 줄어 64.73% 감소했고, 고령자 미완료 횡단도 36.08건에서 13.33건으로 줄어 63.05% 감소했다. 다만 이 지점에서는 차량 평균 지체가 0.88초 증가했고 최대 대기행렬도 0.17대 증가했다.

가장 큰 절대 감소량은 `8430(신당동)`에서 나타났다. `PET_B_surrogate` 고위험은 449.00건에서 310.83건으로 138.17건 줄었다. 고령자 미완료 횡단도 32.33건 감소했다. 이 지점은 안전 개선이 컸지만 차량 평균 지체는 오히려 0.33초 감소했고 최대 대기행렬은 변하지 않았다.

`71666(장충동)`은 baseline에서 PET_B 고위험이 334.92건으로 많았지만, smart 적용 후 감소율은 6.72%로 상대적으로 작았다. 그래도 고령자 미완료 횡단은 5.25건 줄었고, 차량 평균 지체와 최대 대기행렬은 모두 소폭 감소했다.

정리하면, 이번 결과에서 smart 시나리오의 효과는 다음과 같이 볼 수 있다.

- 안전 개선은 `PET_B_surrogate`와 고령자 미완료 횡단에서 뚜렷하다.
- `PET_A_proxy` 개선은 상대적으로 작다.
- 차량 평균 지체와 대기행렬은 전체 평균에서 사실상 증가하지 않았다.
- 효과는 모든 지점에 균일하게 나타나지 않고, PET_B 이벤트가 발생한 일부 지점에 집중되었다.

## 4. 이 Trade-off가 존재하는가?

본 실험에서 trade-off는 다음 조건으로 정의할 수 있다.

```text
smart 적용 후 보행 안전 지표는 개선되지만,
차량 평균 지체 또는 대기행렬이 증가하는 경우
```

이 정의에 따르면 이번 실행에서는 trade-off가 "일부 지점에서 존재하지만, 전체 평균에서는 강하게 나타나지 않았다"고 보는 것이 가장 정확하다.

산점도 `tradeoff_scatter.png`를 보면 실제로 해석 가능한 점은 `PET_B_surrogate` 감소율이 산정된 3개 지점이다. 이 중 `9494`는 오른쪽 위에 위치한다. 즉 PET_B 고위험 감소율이 높고, 동시에 차량 평균 지체도 증가했다. 이 지점은 보행 안전 개선과 차량 지체 증가가 함께 나타난 명확한 local trade-off 사례이다.

반면 `8430`과 `71666`은 왼쪽 위에 위치한다. 두 지점 모두 PET_B 고위험은 감소했지만 차량 평균 지체는 증가하지 않았다. `8430`은 차량 평균 지체가 0.33초 줄었고, `71666`은 1.15초 줄었다. 이 두 지점은 smart 신호가 안전을 개선하면서 차량 평균 지체 비용을 거의 만들지 않은 사례이다.

전체 평균 기준으로도 강한 trade-off는 관측되지 않았다. 12개 지점 평균 차량 지체는 34.98초에서 34.93초로 0.05초 감소했고, 최대 대기행렬 평균은 3.576대로 동일했다. 따라서 "스마트 횡단보도는 안전을 높이는 대신 차량 지체를 크게 증가시킨다"는 식의 강한 결론은 이번 결과로는 지지되지 않는다.

다만 trade-off 구조 자체가 없다고 말하기도 어렵다. 그 이유는 다음과 같다.

- `9494`에서는 안전 개선과 차량 평균 지체 증가가 동시에 발생했다.
- seed 단위 차량 평균 지체 변화는 변동성이 크다. 전체 144개 지점-seed 쌍에서 평균 변화는 -0.05초였지만, 표준편차는 6.22초였고 최솟값은 -35.19초, 최댓값은 +61.39초였다.
- 스마트 신호가 보행 녹색을 연장하는 구조인 이상, 특정 교통량과 신호 위상에서는 차량 녹색 시간이 줄어드는 비용이 발생할 수 있다.
- 이번에는 최대 연장이 1회, 5초로 제한되어 있어 교통 비용이 작게 나타났을 가능성이 있다.

따라서 결론은 다음과 같다.

```text
Trade-off는 존재한다. 다만 이번 60분 x 12 seed 실행에서는
전체 평균 수준의 강한 trade-off가 아니라,
특정 지점에서 나타나는 국지적이고 조건부인 trade-off로 관측되었다.
```

정책적으로는 `9494`처럼 안전 개선이 크지만 차량 지체도 약간 증가하는 지점은 추가 검토가 필요하다. 반면 `8430`처럼 안전 개선이 크고 차량 지체 증가가 없는 지점은 우선 도입 후보로 볼 수 있다. `71666`은 baseline 위험은 크지만 감소율이 낮으므로, 단순 5초 1회 연장보다 더 강한 신호 전략이나 다른 보행 안전 대책을 함께 검토할 필요가 있다.

## 5. 한계

첫째, PET 지표의 한계가 크다. 본 결과의 `PET_A_proxy`와 `PET_B_surrogate`는 실제 현장 trajectory 기반 PET가 아니다. 실제 PET는 상충구역을 떠난 시각과 다음 도로 이용자가 동일 상충구역에 진입한 시각의 차이를 정밀 trajectory pair로 계산해야 한다. 이번 모델의 PET는 SUMO edge, 신호 전환 시점, 보행자 위치와 속도를 이용한 proxy/surrogate이므로 절대적인 사고위험 값으로 해석하면 안 된다.

둘째, `PET_B_surrogate`는 스마트 신호 로직과 매우 가깝게 연결된 지표이다. 보행 녹색 종료 시점의 잔류 보행자를 줄이는 데 초점을 둔 모델이므로, 당연히 `PET_B_surrogate`와 고령자 미완료 횡단에서 효과가 크게 나타난다. 반대로 차량-보행자 상충 전반을 나타내는 `PET_A_proxy`의 개선은 작았다. 따라서 안전 효과를 더 넓게 검증하려면 실제 충돌 후보 pair, TTC, DRAC, 보행자 delay, 차량 yield behavior 같은 추가 surrogate safety measure가 필요하다.

셋째, 교통량과 보행량은 실제 관측값이 아니라 추정 수요이다. 차량 수요는 `추정AADT`를 기반으로 seed별 노이즈를 준 뒤 `seoul_commute_peak` 배율을 적용했고, 보행자 수요는 650~1100명/시 범위의 지수분포 도착으로 생성했다. 이는 첨두 상황을 만들기 위한 실험용 수요이지, 지점별 실제 시간대별 관측 교통량은 아니다. 또한 각 run 내부에서 수요 강도가 시간에 따라 크게 변하지 않으므로, 실제 출근 첨두의 5분 단위 파동, 버스 정차, 군집 보행, 행사성 수요는 반영되지 않는다.

넷째, 네트워크와 신호 모델도 단순화되어 있다. 이번 실행은 기존 OSM/SUMO 네트워크를 재사용했기 때문에 재현성은 높지만, 실제 차로 구성, 회전류, 횡단보도 위치, 보행섬, 주변 교차로 연동 신호가 모두 정확하다고 보장할 수 없다. 횡단보도 길이도 `LANES x 3.5m`로 추정했다. 특히 후보 중 `23040`과 `9494`는 원자료에 결측 보정이 들어간 `nan_flag=True` 지점이므로, 차로수나 제한속도 보정값에 민감할 수 있다.

다섯째, 스마트 센서와 제어 로직은 현실보다 단순하다. 본 모델은 보행자가 있으면 잔여 10초 이하에서 5초를 최대 1회 연장하고, 센서 false negative 5%만 반영한다. 실제 시스템에서는 오검지, 중복 검지, 우천/야간 성능, 보행자 의도 판단, 교통신호 제어기 제약, 주변 교차로 연동, 긴급차량 우선신호 같은 요소가 추가된다.

여섯째, 유효 표본이 제한적이다. 전체 후보는 12개이지만 `PET_B_surrogate` 고위험 감소율이 정의된 지점은 3개뿐이다. 따라서 trade-off 산점도에서 실제 관계를 통계적으로 추정하기에는 표본이 작다. 이번 결과는 "가능한 패턴"을 보여주는 실험 결과이지, 일반화된 정책 효과 추정으로 보기에는 부족하다.

일곱째, seed는 12개로 이전 4개 seed 실행보다 안정적이지만, 여전히 신뢰구간이나 통계 검정이 포함되어 있지 않다. 특히 차량 평균 지체 변화는 seed별 변동성이 커서, 평균값만으로 비용이 없다고 단정하면 안 된다. 향후 분석에서는 seed 수를 늘리고, 지점별 paired test 또는 bootstrap confidence interval을 함께 제시하는 것이 좋다.

종합하면, 이번 실행은 스마트 횡단보도가 보행 녹색 종료 시점의 잔류 위험을 줄일 수 있음을 보여준다. 그러나 PET가 proxy/surrogate이고, 수요와 네트워크가 추정 기반이며, 실제 trade-off가 관측된 유효 지점 수가 작기 때문에 결과는 정책 결론보다는 보고서용 시뮬레이션 근거로 사용하는 것이 적절하다.
