# 스마트 횡단보도 SUMO 시뮬레이션 결과 분석 보고서

## 0. 이 시뮬레이션의 개요

본 시뮬레이션은 서울 중구 내 위험도 상위 횡단보도를 대상으로 일반 신호(`baseline`)와 스마트 횡단보도 신호(`smart`)를 비교한 SUMO 기반 미시 교통 시뮬레이션이다. 목적은 스마트 신호가 보행자 안전 지표를 개선하는지, 그리고 그 과정에서 차량 지체나 대기행렬 증가가 발생하는지를 정량적으로 확인하는 것이다.

분석 대상은 위험도 점수 기준 상위 12개 횡단보도이다. 각 횡단보도에 대해 seed 4개(`42`, `43`, `44`, `45`)를 사용했고, 각 seed마다 baseline과 smart 시나리오를 모두 실행했다. 따라서 총 실행 수는 다음과 같다.

```text
12개 횡단보도 x 4개 seed x 2개 시나리오 = 96회 SUMO 실행
```

실행 조건은 다음과 같다.

| 항목 | 값 |
|---|---:|
| 분석 대상 | 서울 중구 위험도 상위 12개 횡단보도 |
| 시나리오 | baseline, smart |
| seed | 42, 43, 44, 45 |
| 시뮬레이션 분석 구간 | 1800초, 30분 |
| 워밍업 | 300초, 5분 |
| TraCI step length | 0.5초 |
| 총 SUMO 실행 수 | 96회 |
| 실패 케이스 | 없음 |

사용한 주요 지표는 다음과 같다.

| 지표 | 의미 |
|---|---|
| `PET_A_proxy_severe` | 차량이 상충구역 주변을 빠져나간 뒤 보행자가 진입하는 시간차 기반 proxy 위험 건수 |
| `PET_B_surrogate_severe` | 보행 녹색 종료 시점의 잔여 횡단시간과 전적색 시간을 비교한 surrogate 고위험 건수 |
| `elderly_incomplete_cross` | 보행 녹색 종료 시점에 횡단을 완료하지 못한 고령 보행자 수 |
| `veh_avg_delay_sec` | 차량별 누적 대기시간 평균 |
| `veh_max_delay_sec` | 차량별 누적 대기시간 최대값 |
| `queue_avg` | 접근 차로 정지 차량 수 평균 |
| `queue_max` | 접근 차로 정지 차량 수 최대값 |

주의할 점은 PET가 실제 현장 trajectory pair 기반 PET가 아니라는 것이다. 본 모델의 PET 계열 값은 SUMO/TraCI 상태를 이용한 `proxy` 또는 `surrogate` 지표이며, 실제 사고 위험 그 자체가 아니라 상대 비교용 안전 지표로 해석해야 한다.

## 1. 실행방식, 왜 이렇게 했는지

### 1.1 실행방식

실행은 보고서용 프리셋인 `scripts/run_report_simulation.py`로 수행했다. 이 스크립트는 기존에 생성된 OSM/SUMO 네트워크를 결과 폴더 내부로 복사한 뒤, 해당 복사본에서 수요 생성, baseline/smart 실행, 결과 집계, 보고서 생성까지 수행한다.

실행 명령은 다음과 같다.

```bash
python3 scripts/run_report_simulation.py
```

기본 프리셋은 다음과 같다.

| 설정 | 값 |
|---|---:|
| `top_n` | 12 |
| `seeds` | 42, 43, 44, 45 |
| `sim_duration` | 1800 |
| `warmup` | 300 |
| `traci_step_length` | 0.5 |
| `skip_networks` | true |
| 네트워크 방식 | 기존 네트워크 복사 후 재사용 |

### 1.2 위험도 상위 12개만 선택한 이유

전체 횡단보도를 모두 실험하면 계산 비용이 커지고, 보고서에서 해석해야 할 결과도 과도하게 많아진다. 따라서 사고건수, 노인비율, 차로수, 제한속도 등을 반영한 위험도 점수(`risk_score`)를 기준으로 상위 12개 지점을 선정했다.

이 방식의 장점은 다음과 같다.

- 스마트 횡단보도 도입 필요성이 높은 지점을 우선 분석할 수 있다.
- 제한된 시간 안에 보고서용 결과를 안정적으로 생성할 수 있다.
- 지점별 비교와 정책 후보 분류가 가능하다.

### 1.3 4개 seed를 사용한 이유

단일 seed만 사용할 경우 특정 난수 조건에 결과가 과도하게 의존할 수 있다. 본 실험은 seed 4개를 사용하여 각 지점과 시나리오의 평균 결과를 계산했다.

seed 반복의 목적은 다음과 같다.

- 차량 route 생성의 우연성을 줄인다.
- 보행자 도착 시점의 우연성을 줄인다.
- baseline과 smart를 동일 seed 조건에서 비교하여 시나리오 차이만 확인한다.

다만 seed 4개는 보고서용 비교에는 충분하지만, 논문 수준의 통계 검정에는 아직 작다. 정책 결론을 내리려면 seed 수를 10개 이상, 가능하면 30개 이상으로 늘리는 것이 적절하다.

### 1.4 30분 분석 구간과 5분 워밍업을 사용한 이유

분석 구간은 30분(`1800초`)으로 설정했다. 이는 보고서용 산출물을 현실적인 시간 안에 생성하기 위한 타협이다. 워밍업 5분(`300초`)은 시뮬레이션 시작 직후 차량과 보행자가 네트워크에 충분히 진입하지 않은 초기 상태를 제외하기 위한 장치이다.

30분 분석의 장점은 실행 시간이 짧고 여러 seed를 반복하기 쉽다는 점이다. 반면 긴 시간대의 누적 혼잡, 첨두시간 지속 효과, 장시간 대기행렬 전파는 충분히 반영하지 못할 수 있다.

### 1.5 TraCI step length를 0.5초로 둔 이유

기존 기본값은 0.1초였지만, 보고서용 프리셋에서는 `traci_step_length=0.5`를 사용했다. 이유는 실행 시간과 분석 정밀도 사이의 균형이다.

0.1초 step은 더 세밀하지만 96회 실행에서는 시간이 길어진다. 반면 0.5초 step은 신호 전환과 보행자 잔류 여부를 추적하기에는 충분한 수준이면서 실행 시간을 줄일 수 있다. 본 결과는 보고서용 비교 분석이 목적이므로 0.5초 step을 사용했다.

### 1.6 기존 네트워크를 복사해 재사용한 이유

OSM 다운로드와 `netconvert`를 매번 다시 수행하면 외부 API 상태, 네트워크 응답, OSM 데이터 변경에 따라 결과가 달라질 수 있다. 따라서 기존에 안정적으로 생성된 네트워크를 결과 폴더 내부 `sumo_nets/`로 복사하여 사용했다.

이 방식의 장점은 다음과 같다.

- 네트워크 구조를 고정하여 baseline/smart 비교의 일관성을 확보한다.
- 외부 OSM API 상태에 따른 실패 가능성을 줄인다.
- 결과 폴더만으로 실행 조건과 네트워크 파일을 함께 보존할 수 있다.

## 2. Trade-off의 실존에 대한 분석

### 2.1 Trade-off의 정의

본 실험에서 trade-off는 보행자 안전 개선과 차량 운영 비용 증가 사이의 관계로 정의한다.

안전 개선은 다음 지표로 판단한다.

- `PET_B_surrogate_severe` 감소
- `elderly_incomplete_cross` 감소

차량 운영 비용은 다음 지표로 판단한다.

- `veh_avg_delay_sec` 증가
- `queue_max` 증가

즉 smart 시나리오에서 보행자 안전 지표가 개선되지만 차량 평균 지체나 대기행렬이 증가한다면 trade-off가 존재한다고 볼 수 있다.

### 2.2 전체 평균 기준 trade-off

전체 12개 지점의 seed 평균 결과를 시나리오별로 비교하면 다음과 같다.

| 시나리오 | PET_B 고위험 평균 | 고령자 미완료 평균 | 차량 평균 지체 평균(초) | 최대 대기행렬 평균 |
|---|---:|---:|---:|---:|
| baseline | 4.312 | 1.438 | 15.975 | 2.062 |
| smart | 1.646 | 0.396 | 15.967 | 2.146 |
| 변화량 | -2.667 | -1.042 | -0.008 | +0.083 |

전체 평균 기준으로 보면 안전 지표는 뚜렷하게 개선되었다. `PET_B_surrogate_severe`는 평균 4.312건에서 1.646건으로 줄었고, 고령자 미완료 횡단은 평균 1.438건에서 0.396건으로 줄었다.

반면 차량 평균 지체는 15.975초에서 15.967초로 거의 변하지 않았다. 오히려 평균값만 보면 0.008초 감소했다. 최대 대기행렬은 2.062대에서 2.146대로 0.083대 증가했지만, 증가폭은 매우 작다.

따라서 전체 평균 기준 결론은 다음과 같다.

```text
이번 실행에서는 안전 개선은 뚜렷했지만, 차량 평균 지체 증가라는 강한 trade-off는 관측되지 않았다.
```

### 2.3 총량 기준 안전 개선

12개 지점의 seed 평균값을 합산하면 안전 개선은 더 분명하게 나타난다.

| 지표 | baseline 합계 | smart 합계 | 변화량 | 변화율 |
|---|---:|---:|---:|---:|
| PET_A proxy 고위험 | 125.50 | 124.75 | -0.75 | -0.6% |
| PET_B surrogate 고위험 | 51.75 | 19.75 | -32.00 | -61.8% |
| 고령자 미완료 횡단 | 17.25 | 4.75 | -12.50 | -72.5% |

`PET_A_proxy`는 거의 변하지 않았다. 이는 스마트 신호가 차량과 보행자의 상충구역 통과 시점 간격 전체를 크게 바꾸지는 않았다는 뜻이다.

반면 `PET_B_surrogate`와 고령자 미완료 횡단은 크게 줄었다. 이는 본 스마트 신호 로직이 특히 "보행 녹색 종료 시점에 아직 횡단 중인 보행자" 문제를 줄이는 데 효과적이었다는 뜻이다.

### 2.4 지점별 trade-off

PET_B 고위험 이벤트가 baseline에서 실제로 발생한 지점은 3개였다. 이 3개 지점에서만 감소율이 정의된다.

| 횡단보도ID | 행정동 | PET_B 고위험 변화 | 감소율 | 고령자 미완료 변화 | 감소율 | 차량 지체 변화 | 최대 대기행렬 변화 |
|---:|---|---:|---:|---:|---:|---:|---:|
| 8430 | 신당동 | 23.50 -> 6.25 | 73.4% 감소 | 8.50 -> 2.00 | 76.5% 감소 | +0.090초 | +0.25대 |
| 71666 | 장충동 | 21.75 -> 9.50 | 56.3% 감소 | 6.75 -> 1.50 | 77.8% 감소 | -0.008초 | +0.75대 |
| 9494 | 신당동 | 6.50 -> 4.00 | 38.5% 감소 | 2.00 -> 1.25 | 37.5% 감소 | -0.182초 | 0.00대 |

이 표에서 trade-off가 가장 명확한 지점은 `8430`이다. 안전 지표가 크게 개선되었지만 차량 평균 지체가 0.090초 증가했고 최대 대기행렬도 0.25대 증가했다. 다만 증가폭이 작기 때문에 실질적 비용은 제한적이다.

`71666`은 안전 지표가 크게 개선되었고 차량 평균 지체는 오히려 0.008초 감소했다. 그러나 최대 대기행렬이 0.75대 증가했다. 따라서 평균 지체 기준 trade-off는 약하지만, 대기행렬 기준의 운영 비용은 존재한다.

`9494`는 안전 지표가 개선되면서 차량 지체도 감소했다. 이 지점에서는 trade-off라기보다는 smart 시나리오가 안전과 차량 운영 측면에서 동시에 유리하게 나온 경우다.

### 2.5 Trade-off의 실존 여부

이번 결과만 놓고 보면 trade-off는 "강하게 실존한다"고 단정하기 어렵다. 이유는 차량 평균 지체가 전체 평균에서 증가하지 않았고, 안전 개선이 관측된 3개 지점 중 2개 지점에서는 차량 평균 지체가 감소했기 때문이다.

하지만 trade-off의 가능성은 일부 지표에서 관측된다.

- `8430`에서는 안전 개선과 차량 지체 증가가 동시에 발생했다.
- `71666`에서는 안전 개선과 최대 대기행렬 증가가 동시에 발생했다.
- 전체 평균 최대 대기행렬은 2.062대에서 2.146대로 증가했다.
- seed 단위 변화량에서 차량 평균 지체 변화의 표준편차는 1.177초로, 개별 seed에서는 차량 비용이 커질 수 있다.
- seed 단위 차량 평균 지체 변화의 최댓값은 +6.047초, 최솟값은 -5.230초였다.

따라서 결론은 다음과 같다.

```text
보행 안전 개선과 차량 운영 비용 사이의 trade-off 구조는 존재한다.
다만 이번 30분 x 4 seed 결과에서는 차량 평균 지체 비용이 강하게 나타나지 않았고,
대기행렬 증가와 일부 지점의 미세한 지체 증가 형태로 약하게 관측되었다.
```

이 결과는 스마트 신호가 항상 차량 흐름을 크게 악화시킨다는 주장을 뒷받침하지 않는다. 오히려 본 조건에서는 보행자 안전 개선이 차량 평균 지체 증가 없이 달성될 가능성을 보여준다. 다만 장시간 시뮬레이션, 더 높은 교통량, 더 긴 신호 연장 조건에서는 trade-off가 더 강하게 나타날 수 있다.

## 3. 최종 결과 분석

### 3.1 전체 결과 요약

전체적으로 smart 시나리오는 `PET_B_surrogate`와 고령자 미완료 횡단을 크게 줄였다.

| 지표 | baseline | smart | 변화량 | 해석 |
|---|---:|---:|---:|---|
| PET_A proxy 고위험 평균 | 10.458 | 10.396 | -0.062 | 거의 변화 없음 |
| PET_B surrogate 고위험 평균 | 4.312 | 1.646 | -2.667 | 뚜렷한 개선 |
| 고령자 미완료 횡단 평균 | 1.438 | 0.396 | -1.042 | 뚜렷한 개선 |
| 차량 평균 지체 평균 | 15.975초 | 15.967초 | -0.008초 | 사실상 변화 없음 |
| 차량 최대 지체 평균 | 56.333초 | 56.354초 | +0.021초 | 사실상 변화 없음 |

| 최대 대기행렬 | 2.062대 | 2.146대 | +0.083대 | 매우 작은 증가 |

핵심은 `PET_B_surrogate`와 고령자 미완료 횡단에서 개선이 집중되었다는 점이다. 이는 현재 smart 로직이 보행자 잔류 상황에서 보행 녹색을 연장하는 방식이기 때문이다. 즉 스마트 신호는 모든 안전지표를 균일하게 개선하기보다는, 신호 종료 시점의 보행자 잔류 위험을 줄이는 데 효과가 있었다.

### 3.3 우선 도입 후보

가장 강한 개선을 보인 지점은 `8430(신당동)`이다.

`8430`의 결과는 다음과 같다.

| 지표 | baseline | smart | 변화 |
|---|---:|---:|---:|
| PET_B 고위험 | 23.50 | 6.25 | 73.4% 감소 |
| 고령자 미완료 횡단 | 8.50 | 2.00 | 76.5% 감소 |
| 차량 평균 지체 | 5.596초 | 5.686초 | +0.090초 |
| 최대 대기행렬 | 5.00대 | 5.25대 | +0.25대 |

이 지점은 안전 개선 폭이 가장 크고 차량 지체 증가는 0.090초에 불과하다. 따라서 본 실험 조건에서는 가장 우선적으로 스마트 횡단보도 도입을 검토할 수 있는 후보이다.

### 3.5 개선 효과는 있으나 추가 검토가 필요한 후보

`9494(신당동)`은 안전 개선과 차량 지체 감소가 동시에 관측되었다.

| 지표 | baseline | smart | 변화 |
|---|---:|---:|---:|
| PET_B 고위험 | 6.50 | 4.00 | 38.5% 감소 |
| 고령자 미완료 횡단 | 2.00 | 1.25 | 37.5% 감소 |
| 차량 평균 지체 | 6.621초 | 6.439초 | -0.182초 |
| 최대 대기행렬 | 0.75대 | 0.75대 | 0.00대 |

이 지점은 smart 시나리오가 안전과 차량 운영 모두에서 유리하게 나왔다. 다만 개선 폭은 `8430`, `71666`보다 작다. 따라서 우선 도입 후보라기보다는 추가 반복 실험을 통해 효과의 일관성을 확인할 후보로 보는 것이 적절하다.

### 3.6 이벤트 미관측 지점

나머지 9개 지점은 `PET_B_surrogate_severe`와 고령자 미완료 횡단이 baseline과 smart 모두 0으로 나타났다.

이 결과는 세 가지 방식으로 해석할 수 있다.

- 실제로 본 수요 조건에서는 신호 종료 시점의 보행자 잔류 위험이 작았을 수 있다.
- 30분 x 4 seed 실험이 해당 위험 이벤트를 충분히 발생시키지 못했을 수 있다.
- OSM/SUMO 네트워크의 crossing 구조나 보행자 route가 해당 위험을 잘 드러내지 못했을 수 있다.

특히 `6213`은 차량 평균 지체가 약 69.44초로 매우 높았지만 PET_B 이벤트는 발생하지 않았다. 이는 차량 혼잡이 높다고 해서 보행자 잔류 위험이 자동으로 발생하는 것은 아니라는 점을 보여준다. 차량 혼잡과 보행자 안전 위험은 별도 지표로 봐야 한다.

### 3.8 최종 결론

이번 보고서용 시뮬레이션의 핵심 결론은 다음과 같다.

첫째, smart 시나리오는 `PET_B_surrogate` 고위험과 고령자 미완료 횡단을 크게 줄였다. PET_B 고위험 총량은 51.75에서 19.75로 61.8% 감소했고, 고령자 미완료 횡단 총량은 17.25에서 4.75로 72.5% 감소했다.

둘째, 차량 평균 지체는 거의 증가하지 않았다. 전체 평균 기준 차량 지체는 15.975초에서 15.967초로 사실상 동일했다. 따라서 이번 조건에서는 스마트 신호가 보행자 안전을 개선하면서도 차량 평균 지체를 크게 악화시키지 않았다.

셋째, trade-off는 약하게 존재한다. `8430`에서는 안전 개선과 차량 지체 증가가 동시에 나타났고, `71666`에서는 안전 개선과 최대 대기행렬 증가가 함께 나타났다. 그러나 전체 평균에서는 차량 지체 trade-off가 강하게 나타나지 않았다.

넷째, 정책 후보로는 `8430`이 가장 우선적이다. 안전 개선 폭이 가장 크고 차량 지체 증가는 0.090초로 작다. `71666`은 조건부 도입 후보이며, `9494`는 추가 반복 실험을 통해 효과의 일관성을 검토할 후보이다.

다섯째, 본 결과는 보고서와 발표용 비교 분석에는 충분하지만, 실제 정책 결론을 위해서는 추가 실험이 필요하다. 특히 seed 수 확대, 실제 보행자·차량 수요 보정, 실제 trajectory 기반 PET 계산, 장시간 시뮬레이션이 필요하다.
